# Programming style

- 代碼的寫法應該使理解代碼的人所需要的時間最小化
- 就算是自己一個人開發的專案也需要留意代碼的可讀性，因為一段時間之後自己就很可能會忘記裡面的內容
- 一般而言代碼的行數越少越好，但仍需要考量程式碼的可讀性

- 要編寫易於被人理解的程式，但還需要考慮可讀性才能決定程式的寫法

- 多考慮如果是別人要來閱讀我們的程式碼會不會需要額外的時間

- 盡量把程式的內容訊息裝進變數的名稱中

因為代碼是給人看的不是機器

- 容易被理解
- 簡潔
